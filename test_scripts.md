1. Базовый сценарий работы системы

Запуск сервера
Подключение клиента к серверу
Отправка логов различных уровней (DEBUG, INFO, WARNING, ERROR, FATAL)
Сохранение логов в файл
Корректное завершение работы клиента и сервера

2. Ротация файлов по размеру

Заполнение текущего лог-файла до достижения лимита размера (15 КБ)
Автоматическое создание нового файла с суффиксом _001, _002 и т.д.
Продолжение записи логов в новый файл

3. Фильтрация логов в реальном времени

Включение режима Live View
Установка фильтра по уровню (INFO, WARNING, ERROR, FATAL)
Проверка, что в консоль и файл попадают только логи соответствующего уровня
Сброс фильтра (показ всех логов)

4. Обработка критических ошибок (FATAL + stackTrace)

Отправка лога уровня FATAL со stackTrace
Корректное сохранение stackTrace в файл
Проверка формата записи

5. Работа с несколькими клиентами

Запуск сервера
Подключение нескольких клиентов одновременно
Отправка логов от разных клиентов
Агрегация всех логов на сервере

6. Тестирование разных сервисов

Отправка логов от разных сервисов (AuthService, DatabaseService, PaymentService, Monitoring и т.д.)
Проверка корректности записи service name

7. Тестирование JSON-формата

Отправка лога клиентом
Проверка, что сервер правильно парсит JSON
Корректное сохранение всех полей (timestamp, requestId, level, service, message, stackTrace)


8. Устойчивость к отключениям

Подключение клиента
Отправка логов
Резкое отключение клиента
Корректное продолжение работы сервера

9. Производительность при большой нагрузке

Отправка большого количества логов (100+)
Проверка стабильности ротации и отсутствия утечек памяти
